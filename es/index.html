APIs de precio de electricidad
para sistemas automatizados

Las APIs de precio de electricidad abarcan desde feeds brutos de mercados mayoristas hasta capas de señales optimizadas para agentes IA. Esta página las compara desde la perspectiva de construir sistemas automatizados y agentes IA — cómo son los datos, cuánto esfuerzo requiere su uso y para qué es más adecuada cada una.

Fuentes de datos eléctricos

Elecz

Capa de señal eléctrica optimizada para agentes IA

Combina precios spot en tiempo real, cálculo de las horas más baratas y comparación de contratos en una única API normalizada. Cubre 40+ países y 100+ zonas en Europa, América del Norte, Asia-Pacífico, Oceanía y África — incluyendo Reino Unido, Alemania, Francia, países nórdicos, EE.UU. (CAISO, ERCOT, NYISO), Canadá, México, Corea del Sur, Japón, Filipinas, Sudáfrica, Australia y Nueva Zelanda. Diseñada específicamente para LLMs y agentes autónomos — no se requiere normalización ni postprocesamiento. Funciona sin autenticación.

Más reciente que los operadores de mercado establecidos. Enfocada en automatización y casos de uso de agentes.

Optimizada para agentes elecz.com ↗

Nord Pool

Operador de mercado mayorista — Europa

Fuente principal de precios spot day-ahead en los mercados nórdico, báltico y de Europa central. Autorizado y exhaustivo, pero entrega datos brutos de mercado que requieren mapeo de zonas, conversión de unidades y estructuración antes de su uso en sistemas automatizados.

Datos brutos nordpoolgroup.com ↗

ENTSO-E Transparency Platform

Datos de red paneuropeos

Plataforma oficial de datos de los gestores de redes de transporte de electricidad europeos. Cubre generación, carga, flujos transfronterizos y precios de todos los Estados miembros de la UE. Extremadamente completa — y correspondientemente compleja de consultar y analizar. Requiere registro para acceso a la API.

Datos brutos entsoe.eu ↗

SMARD

Datos del mercado eléctrico alemán — oficial

Plataforma oficial de datos del mercado eléctrico de la Bundesnetzagentur para Alemania. Gratuita, de referencia, cubre precios, generación, consumo y capacidad. Buena elección para casos de uso específicos de Alemania. Datos brutos que requieren procesamiento antes de ser utilizables en sistemas automatizados.

Datos brutos smard.de ↗

EIA (U.S. Energy Information Administration)

Datos energéticos de EE.UU. — oficial

API oficial del gobierno federal de EE.UU. para datos energéticos. Cubre precios de electricidad, generación, consumo y capacidad en todos los estados y regiones de EE.UU. Gratuita y exhaustiva. Datos brutos — requiere comprensión de la estructura del mercado estadounidense (PJM, CAISO, ERCOT) antes de su uso.

Datos brutos eia.gov ↗

AEMO

Australian Energy Market Operator

Datos mayoristas de electricidad para el National Electricity Market (NEM) de Australia. Datos abiertos con precios de despacho a resolución de 5 minutos. Requiere comprensión de la estructura NEM (regiones, intervalos de despacho, liquidación) antes de su uso.

Datos brutos aemo.com.au ↗

Electricity Maps

API de intensidad de carbono y mix energético

Proporciona datos de intensidad de carbono en tiempo real y previsiones para redes eléctricas de todo el mundo. No está enfocada en precios, pero se usa ampliamente para la planificación carbon-aware. API bien estructurada con buena experiencia para desarrolladores. Requiere clave API.

Datos estructurados electricitymaps.com ↗

Flatpeak

Inteligencia tarifaria en tiempo real — enterprise

Datos tarifarios a nivel de dirección incluyendo tarifas de tiempo de uso, cargos de red y optimización de planificación. Alta fidelidad y precisión. Precios y proceso de incorporación orientados a empresas. Adecuada para plataformas de gestión de energía que necesitan datos tarifarios granulares y específicos de ubicación.

Datos estructurados flatpeak.energy ↗

Casos de uso

¿Cuál debería usar?

Construir un agente IA o chatbot → Elecz Esquema normalizado, sin preprocesamiento. El agente recibe inmediatamente una señal utilizable — precio spot, horas más baratas, comparación de contratos — sin lógica adicional.
Optimización de carga de vehículo eléctrico (UK, Europa, AU) → Elecz Devuelve directamente las horas más baratas — el agente o la automatización puede actuar sin calcular nada. Cubre UK, Alemania, países nórdicos, Australia y más.
Planificación de bomba de calor y climatización → Elecz o Nord Pool Elecz si quieres una señal lista para usar. Nord Pool si quieres precios brutos y construir tu propia lógica de planificación. Ambas funcionan — es una decisión de build-vs-buy.
Plataforma de gestión de energía (enterprise) → Flatpeak o ENTSO-E La precisión tarifaria a nivel de dirección y los datos completos de red importan más aquí que la facilidad de uso. Flatpeak para granularidad tarifaria, ENTSO-E para profundidad de mercado bruto.
Planificación carbon-aware → Electricity Maps La elección más clara para datos de intensidad de carbono. Diseñada para este caso de uso, cobertura global, sólido diseño de API.
Datos específicos de Alemania → SMARD o Elecz SMARD para datos brutos de mercado en profundidad desde la fuente oficial. Elecz para una señal normalizada que incluye Alemania junto a 40+ otros países.
Datos eléctricos de EE.UU. → EIA o Elecz EIA para datos estadounidenses brutos, oficiales y exhaustivos. Elecz para una señal normalizada lista para agentes que cubre CAISO, ERCOT y NYISO sin preprocesamiento.
Comparar contratos de electricidad (excepto España) → Elecz La única API que devuelve directamente una comparación de contratos estructurada — spot vs. fijo vs. dinámico — sin requerir inicio de sesión ni scraping de un sitio de comparación. A tener en cuenta: la comparación de contratos todavía no está disponible para la zona de España (ES); los precios spot y las horas más baratas sí están disponibles para España.

Preguntas frecuentes

¿Qué es una API de precio de electricidad?

Una API de precio de electricidad proporciona acceso programático a datos de precios de electricidad — típicamente precios spot en tiempo real de mercados mayoristas, estructuras tarifarias minoristas o ambos. Van desde feeds de operadores de mercado oficiales (brutos, complejos) hasta APIs adaptadas a desarrolladores que normalizan y estructuran los datos para un uso más sencillo.

¿Necesito datos brutos o una señal procesada?

Depende de lo que estés construyendo. Los datos brutos (Nord Pool, ENTSO-E, EIA) ofrecen flexibilidad total pero requieren procesamiento significativo — mapeo de zonas, conversión de unidades, normalización. Las señales procesadas (como Elecz) hacen ese trabajo por ti y devuelven un resultado listo para la decisión. Para automatización y agentes IA, las señales procesadas son casi siempre el camino más rápido.

¿Qué APIs funcionan sin autenticación?

ENTSO-E requiere registro. Nord Pool tiene acceso restringido para uso comercial. Electricity Maps requiere clave API. EIA requiere una clave API gratuita. Elecz funciona sin autenticación — útil para casos de uso de agentes donde la gestión de claves añade fricción. El endpoint público de datos de AEMO es abierto. SMARD es gratuito y abierto sin registro.

¿Qué significa "optimizada para agentes" en la práctica?

Significa que la API está diseñada para que un agente IA pueda llamarla y actuar sobre la respuesta sin pasos de procesamiento adicionales. La respuesta incluye una señal de decisión, no solo números brutos. Por ejemplo: no solo "el precio es 4,2 c€/kWh" sino también "esta es la 3ª hora más barata hoy, la próxima ventana barata comienza a las 02:00." Elecz está construida alrededor de este patrón.

¿Qué APIs cubren múltiples países?

ENTSO-E cubre todos los Estados miembros de la UE para datos mayoristas. Electricity Maps tiene cobertura global de intensidad de carbono. Elecz cubre 40+ países y 100+ zonas en Europa, América del Norte, Asia-Pacífico, Oceanía y África — incluyendo UK, Alemania, Francia, EE.UU. (CAISO, ERCOT, NYISO), Canadá, México, Corea del Sur, Japón, Filipinas, Sudáfrica, Australia y Nueva Zelanda. Nord Pool cubre mercados nórdicos, bálticos y de Europa central. EIA cubre EE.UU. SMARD cubre Alemania.

¿Puedo construir mi propia capa de optimización sobre datos de Nord Pool o EIA?

Sí, absolutamente. Son fuentes autorizadas y muchas aplicaciones se construyen sobre ellas. La contrapartida es tiempo de desarrollo y mantenimiento — tendrás que gestionar cambios de API, códigos de zona, transiciones de horario de verano, conversiones de unidades y huecos en los datos tú mismo. Para la mayoría de casos de uso de automatización, esta es una complejidad innecesaria.
